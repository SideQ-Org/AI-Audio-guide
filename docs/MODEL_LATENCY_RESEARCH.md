# Ресёрч: мгновенные ответы на вопросы — модели и задержки (2026-07)

Задача: снизить задержку ответа на голосовой вопрос (barge-in) и оценить предложение Mell —
перейти на **мультимодальную голосовую модель GPT (Realtime API, voice-to-voice)**, а также
сравнить с другими вариантами из OpenRouter для «two-tier» ответа (быстрая модель отвечает мгновенно,
сильная усиливает).

> Цены/ID моделей меняются ежемесячно — часть цифр из вторичных источников, помечено. Проверяйте на
> официальных страницах перед бюджетированием. **Источник истины по достижимости — probe-скрипт
> `backend/sim/ttft_probe.py`, запущенный С ПРОД-ХОСТА.**

## Главный вывод (TL;DR)

**Геоблок решает всё.** Прод-регион: OpenRouter отдаёт 403 на модели OpenAI / Google / Anthropic
(и прямые API этих провайдеров тоже заблокированы). Поэтому:

- **GPT Realtime voice (предложение Mell) — недостижим как есть и архитектурно не подходит.**
  OpenRouter **вообще не проксирует** WebSocket Realtime API (только HTTP/SSE chat-completions +
  batch TTS/STT), а OpenAI из региона заблокирован. Нужен **релей вне региона**. Плюс он конфликтует
  с нашим клонированным голосом гида (grok-voice), 8 языками и логикой barge-in/pause/weave. Вердикт:
  интересно, но не сейчас.
- **Gemini Live — та же стена.** Google заблокирован, WebSocket Live тоже не проксируется OpenRouter.
- **Достижимый выигрыш — two-tier ТЕКСТ на open-weight моделях через OpenRouter**: Groq-/Cerebras-
  хостовые Llama/Qwen проксируются серверами OpenRouter и **НЕ попадают** под геоблок OpenAI/Google/
  Anthropic. Это конкретный деплоящийся путь.

## Сравнение

| Вариант | TTFT / латентность | Цена (вторичные источники) | Достижимо из прод-региона? |
|---|---|---|---|
| **GPT Realtime** (`gpt-realtime-2.1`, `-mini`) | ~250 мс voice-to-voice | audio $32/$64 за 1M (кэш in $0.40); mini $10/$20; ~$0.04–0.30/мин | ❌ только через релей вне региона (OpenRouter не проксирует WS) |
| **Gemini Live** (`gemini-3.1-flash-live-preview`) | низкая, native S2S | ~$3/$12 за 1M audio (~$0.005/$0.018 за мин); сессия ≤15 мин | ❌ та же стена |
| **Groq (LPU)** Llama 4 / Qwen 3 / Kimi | **TTFT <0.1–0.18 с**, ~320–800 tok/s | дёшево | ✅ **через OpenRouter** (прямой Groq блокирует регион) |
| **Cerebras (WSE-3)** Llama / Qwen 3 / DeepSeek | TTFT ~0.08–0.16 с, **2100–2900 tok/s** | дёшево | ✅ **через OpenRouter** |
| **DeepSeek** `deepseek-chat` (non-thinking) | текущие ~2 с | дёшево | ✅ уже наш прод-ответчик |
| Фронтир (GPT-5.5 / Claude Opus 4.7 / Gemini 3 Pro) | 0.7–1.4 с | дорого | ❌ геоблок |

Ключевое: геоблок специфичен для **моделей** OpenAI/Google/Anthropic — open-weight на Groq/Cerebras
через OpenRouter доступны. **Не включать reasoning/thinking на быстром тире** (раздувает TTFT в 5–30×).
Целевая метрика — **TTFT, не throughput** (выше ~200–300 tok/s разница неощутима под темп озвучки).

## Рекомендация для нашего приложения

**(a) Мгновенные ответы сейчас — two-tier ТЕКСТ, всё через OpenRouter:**
- **Тир 1 (мгновенный, TTFT <1 с):** маленькая open-модель через Groq — первое предложение/подтверждение.
  Кандидаты: `meta-llama/llama-3.3-70b-instruct` (Groq), `qwen/qwen3-*-instruct` (Groq/Cerebras),
  `moonshotai/kimi-k2` (Groq). Ограничить 1–2 предложениями, без thinking.
- **Тир 2 (усиление, стримом):** текущий **`deepseek/deepseek-chat`** — продолжает/углубляет, не
  переписывает; вплетается на границе предложения через существующий `NarrationScheduler`.
- STT (`voxtral-mini-transcribe`) и TTS (`grok-voice-tts-1.0`) — без изменений, голос и 8 языков
  сохраняются, переиспользуется вся логика barge-in/pause/weave. Минимальный риск.

**(b) Стоит ли GPT Realtime vs текущий STT→LLM→TTS?** Для прода — нет: выигрыш латентности реален
(~250 мс), но **недостижим без релея вне региона**, и он заменил бы наш консистентный голос + 8 языков
+ фактологичность на чёрный ящик с кумулятивным биллингом. Если пробовать — поднять релей-VM вне
региона и прототипировать скорее **Gemini Live** (в ~10× дешевле Realtime), но это R&D-спайк, не
переключение прода. Mell: «революция реальна, но региональная стена делает её релей-хостед R&D».

**(c) Конкретная достижимая пара для внедрения:**
- Fast: **`meta-llama/llama-3.3-70b-instruct` через Groq** (`provider: {order:["Groq"]}`), без thinking, кап 1–2 предложения.
- Strong: **`deepseek/deepseek-chat`** (текущий), стримом, «продолжай, только если добавляешь проверенный факт».
- Фолбэк: **Cerebras-Qwen 3 / Llama** (выше throughput, TTFT ~0.16 с). Проверить все три probe-скриптом.

## Подводные камни two-tier
- **Противоречие:** если сильная модель РЕГЕНЕРИРУЕТ, ответы могут разойтись вслух. Безопасно: (a) быстрый
  тир говорит короткое безопасное первое предложение/подтверждение, сильный ПРОДОЛЖАЕТ; или (b) быстрый
  отвечает целиком, сильный говорит дополнение **только если добавляет факт** (иначе `[SILENCE]` — как
  наша анти-фабрикация).
- **Не передавать черновик слабой модели как «подсказку» сильной** — исследования показывают, это может
  ухудшить ответ. Сильный тир отвечает из тех же фактов/контекста, не из текста черновика.
- **Стримить тир-2**, чтобы его TTFT прятался за уже играющим тиром-1 — идеально ложится на
  sentence-level `NarrationScheduler`.

## Как измерить реально
`backend/sim/ttft_probe.py` — замеряет TTFT по шорт-листу моделей через OpenAI-совместимый
`/chat/completions` (stream). **Запускать с прод-хоста** (иначе геоблок/сеть не воспроизведутся).
Закомментированная строка `openai/gpt-4o-mini` — контроль: должна упасть 403 из заблокированного региона.

## Two-tier УЖЕ реализован — как включить

Механика встроена (быстрый тир → одно предложение → сильный DeepSeek продолжает, не повторяя).
По умолчанию **выключена** (нет быстрой модели). Включение — только `.env`, кода не трогать:

```env
ANSWER_TWO_TIER=true                                  # (по умолчанию true; активируется моделью ниже)
OPENAI_MODEL_ANSWER_FAST=meta-llama/llama-3.3-70b-instruct
OPENAI_PROVIDER_ANSWER_FAST=Groq                      # пин провайдера OpenRouter (можно "Groq,Cerebras")
# OPENAI_MODEL_COMPANION остаётся сильным тиром (deepseek/deepseek-chat)
```

Прогнать `python -m sim.ttft_probe` с прод-хоста, выбрать модель с TTFT <1с, вписать её в
`OPENAI_MODEL_ANSWER_FAST`. Без `OPENAI_MODEL_ANSWER_FAST` тур работает как раньше (один тир).
Реализация: `Role.ANSWER_FAST` (router/client), `LLMCompanion.respond_fast`, `CompanionInput.already_said`
(сильный тир продолжает без повтора), `Connection._answer_streaming` (main.py).
